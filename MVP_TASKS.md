# MVP_TASKS：恐龙侠打BOSS

本文件将 MVP 拆成可执行的开发任务。当前阶段只做任务规划，不包含业务代码。

## 阶段一：项目初始化和课表模块

### 目标

建立可运行的项目骨架，完成移动端优先的基础布局和课表手动录入能力。

### 任务列表

- [ ] 初始化 Next.js + TypeScript 项目。
- [ ] 配置 Tailwind CSS、代码格式化、基础 lint。
- [ ] 配置 Vitest 单元测试环境。
- [ ] 配置 Playwright 端到端测试环境。
- [ ] 建立基础页面结构：首页、课表页、Boss 页、设置页。
- [ ] 建立移动端优先的底部导航。
- [ ] 建立用户 timezone 设置能力。
- [ ] 设计 Course 数据模型。
- [ ] 实现课程手动新增表单。
- [ ] 实现课程编辑和删除。
- [ ] 实现课程时间冲突校验。
- [ ] 实现周课表视图。
- [ ] 实现今日课程列表。
- [ ] 确保课程时间保存为 UTC，前端按用户 timezone 展示。
- [ ] 为课程时间转换和冲突校验补充单元测试。

### 验收标准

- 用户可以手动创建、编辑、删除课程。
- 课程以周视图和今日列表展示。
- 同一时间段课程冲突时有中文提示。
- 数据中的时间字段为 UTC。
- 移动端页面可正常使用。

## 阶段二：Boss 目标和 AI 拆解

### 目标

用户可以创建学习目标，系统通过服务端 AI 将目标拆成 Boss、阶段任务和每日副本建议。

### 任务列表

- [ ] 设计 BossGoal、StageTask、DailyQuest 数据模型。
- [ ] 实现创建目标页面。
- [ ] 支持目标类型：考试、作业、论文、考证、竞赛、其他。
- [ ] 实现 Boss 列表页。
- [ ] 实现 Boss 详情页。
- [ ] 设计 AI 拆解 JSON schema。
- [ ] 实现服务端 AI 调用接口。
- [ ] 确保 OpenAI API Key 只从服务端环境变量读取。
- [ ] 实现 AI 输出 JSON schema 校验。
- [ ] 实现 AI 输出失败时的中文错误提示和重试入口。
- [ ] 将通过校验的 AI 结果保存为 Boss、阶段任务和每日副本草稿。
- [ ] 提供 AI 拆解结果确认界面。
- [ ] 支持用户轻量编辑 AI 生成的任务标题、预计时长和说明。
- [ ] 为 AI JSON schema 校验补充单元测试。

### 验收标准

- 用户输入目标后可以生成 Boss。
- AI 返回内容必须是结构化 JSON。
- 无效 AI 输出不会进入业务数据。
- Boss 详情中能看到阶段任务和每日副本建议。
- 前端无法读取 OpenAI API Key。

## 阶段三：自动排程

### 目标

系统根据课表空闲时间、目标截止日期和任务预计时长，为每日副本生成可执行时间安排。

### 任务列表

- [ ] 定义排程算法输入结构：课程、任务、截止日期、timezone、每日偏好。
- [ ] 定义排程算法输出结构：每日副本、开始时间、结束时间、冲突信息。
- [ ] 实现空闲时间计算。
- [ ] 实现课程时间硬约束，禁止任务覆盖课程。
- [ ] 实现按截止日期和优先级排序任务。
- [ ] 实现任务切分策略，长任务可拆到多个空闲时间段。
- [ ] 实现最小任务时长限制，避免生成过碎的副本。
- [ ] 实现跨天排程。
- [ ] 实现排程失败提示，例如剩余时间不足。
- [ ] 实现生成排程 API。
- [ ] 实现重新排程 API。
- [ ] 实现今日副本 API。
- [ ] 为排程算法补充单元测试：无课程、单课程、多课程冲突、跨天、截止日期不足、timezone 转换、任务逾期。
- [ ] 在 Boss 详情页展示排程结果。

### 验收标准

- 任务不会被排进课程时间。
- 排程结果按 UTC 保存。
- 前端按用户 timezone 展示排程。
- 当剩余空闲时间不足时，系统给出明确中文提示。
- 排程算法关键场景均有单元测试。

## 阶段四：今日副本和游戏化

### 目标

用户每天能看到今日副本，完成后获得战斗反馈，形成学习推进闭环。

### 任务列表

- [ ] 实现首页今日副本列表。
- [ ] 展示今日课程概览。
- [ ] 展示当前主 Boss 血量。
- [ ] 实现副本开始状态。
- [ ] 实现副本完成状态。
- [ ] 完成副本后扣除 Boss 血量。
- [ ] 完成副本后增加用户 XP。
- [ ] 实现 Boss 被击败状态。
- [ ] 实现未完成副本标记。
- [ ] 未完成副本触发自动重排。
- [ ] 未完成副本导致 Boss 轻微回血。
- [ ] 实现每日战斗结算生成。
- [ ] 实现战斗结算页。
- [ ] 实现连续学习天数。
- [ ] 为伤害、XP、回血和结算逻辑补充单元测试。

### 验收标准

- 用户可以在首页完成今日副本。
- 完成副本后 Boss 血量和用户 XP 正确更新。
- 未完成副本会重排，Boss 会轻微回血。
- 每天能看到中文战斗结算。
- 游戏化反馈不阻碍任务执行。

### 目标

提升移动端可用性、错误恢复能力和完整闭环体验，为真实用户试用做准备。

### 任务列表

- [ ] 优化移动端首页信息密度。
- [ ] 优化课表周视图在小屏幕上的展示。
- [ ] 增加空状态中文文案。
- [ ] 增加加载状态和错误状态。
- [ ] 增加目标创建过程中的草稿保存。
- [ ] 增加 AI 拆解结果的手动修正入口。
- [ ] 增加排程不足时的调整建议。
- [ ] 增加今日副本完成后的轻量动效。
- [ ] 增加 PWA 基础能力评估。
- [ ] 增加基础数据埋点方案：创建课程、创建 Boss、完成副本、触发重排。
- [ ] 整理 README 启动方式和环境变量说明。
- [ ] 补充端到端测试：首次录入课表、创建 Boss、完成今日副本、查看结算。
- [ ] 邀请小范围用户试用并记录反馈。

### 验收标准

- 新用户可以在移动端完成首次核心闭环。
- 常见错误都有中文提示和恢复入口。
- 核心路径具备基础端到端测试。
- README 能支持开发者启动项目。
- MVP 可以进入小范围试用。
